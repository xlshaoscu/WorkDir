## 一、nn.Conv2d 数学公式

### 1.1 基本定义

`nn.Conv2d` 执行的是**二维卷积运算**，其数学公式为：

$$
\text{Out}(N_i, C_{out_j}) = \text{Bias}(C_{out_j}) + \sum_{k=0}^{C_{in}-1} \text{Weight}(C_{out_j}, k) \star \text{Input}(N_i, k)
$$

其中：

- $\star$ 表示二维互相关（cross-correlation）运算
- $N_i$ 是 batch 中的第 $i$ 个样本
- $C_{out_j}$ 是第 $j$ 个输出通道
- $C_{in}$ 是输入通道数

### 1.2 展开形式（无偏置情况）

当 `bias=False` 时，公式简化为：

$$
\text{Out}[n, c_{out}, h, w] = \sum_{c_{in}=0}^{C_{in}-1} \sum_{i=0}^{K_H-1} \sum_{j=0}^{K_W-1} \text{Input}[n, c_{in}, h \times S_H + i - P_H, w \times S_W + j - P_W] \times \text{Weight}[c_{out}, c_{in}, i, j]
$$

参数说明：

| 符号               | 含义                | 本例中的值   |
| ---------------- | ----------------- | ------- |
| $N$              | Batch Size (批量大小) | 1       |
| $C_{in}$         | 输入通道数             | 3 (RGB) |
| $C_{out}$        | 输出通道数             | 2       |
| $H_{in}, W_{in}$ | 输入高度、宽度           | 4, 4    |
| $K_H, K_W$       | 卷积核高度、宽度          | 3, 3    |
| $S_H, S_W$       | 步长 (Stride)       | 1, 1    |
| $P_H, P_W$       | 填充 (Padding)      | 1, 1    |

### 1.3 3×3 卷积的滑动窗口计算

对于 **3×3 卷积**，每个输出位置需要计算 9 个位置的加权和（每个输入通道）：

$$
\text{Out}[n, c_{out}, h, w] = \sum_{c_{in}=0}^{C_{in}-1} \sum_{i=0}^{2} \sum_{j=0}^{2} \text{Input}[n, c_{in}, h + i - 1, w + j - 1] \times \text{Weight}[c_{out}, c_{in}, i, j]
$$

注意：$h + i - 1$ 和 $w + j - 1$ 中的 `-1` 是因为 `padding=1`，卷积核中心对齐。

### 2.3 计算步骤详解

#### 步骤 1: 确定输出形状

输出形状计算公式：

$$
H_{out} = \left\lfloor \frac{H_{in} + 2 \times P_H - K_H}{S_H} \right\rfloor + 1 = \left\lfloor \frac{4 + 2 \times 1 - 3}{1} \right\rfloor + 1 = 4
$$

$$
W_{out} = \left\lfloor \frac{W_{in} + 2 \times P_W - K_W}{S_W} \right\rfloor + 1 = \left\lfloor \frac{4 + 2 \times 1 - 3}{1} \right\rfloor + 1 = 4
$$

**输出形状**: `[1, 2, 4, 4]`  
即: `[Batch, 输出通道, 高, 宽]`

#### 步骤 2: 理解 Padding 的作用

当 `padding=1` 时，输入图像四周各填充 1 层 0：

```

├──┼──┼──┼──┤           ├──┼──┼──┼──┼──┼──┤
│ 5│ 6│ 7│ 8│           │ 0│ 1│ 2│ 3│ 4│ 0│
├──┼──┼──┼──┤    →      ├──┼──┼──┼──┼──┼──┤
│ 9│10│11│12│           │ 0│ 5│ 6│ 7│ 8│ 0│
```

#### 步骤 3: 逐个位置计算示例

以 **输出位置 [0, 0, 1, 1]** (Batch=0, OutCh=0, H=1, W=1) 为例：

**确定输入区域**:

- 输出位置 (1, 1) 对应卷积核中心在填充后输入的 (2, 2) 位置
- 卷积核覆盖区域: 填充后输入的 (1~3, 1~3) → 原始输入的 (0~2, 0~2)

**公式**:
$$
\text{Out}[0, 0, 1, 1] = \sum_{c_{in}=0}^{2} \sum_{i=0}^{2} \sum_{j=0}^{2} \text{Input}[0, c_{in}, i, j] \times \text{Weight}[0, c_{in}, i, j]
$$

## 四、3×3 卷积的本质

### 4.1 感受野

3×3 卷积的**感受野**是 3×3 的区域，意味着每个输出值依赖于输入中 3×3 的局部区域。

```
输入 (5×5)                    输出 (5×5, padding=1)
┌──┬──┬──┬──┬──┐             ┌──┬──┬──┬──┬──┐
│  │  │  │  │  │             │○ │  │  │  │  │
├──┼──┼──┼──┼──┤             ├──┼──┼──┼──┼──┤
│  │██│██│██│  │             │  │  │  │  │  │
├──┼──┼──┼──┼──┤             ├──┼──┼──┼──┼──┤
│  │██│██│██│  │  ──────→    │  │  │● │  │  │
├──┼──┼──┼──┼──┤             ├──┼──┼──┼──┼──┤
│  │██│██│██│  │             │  │  │  │  │  │
├──┼──┼──┼──┼──┤             ├──┼──┼──┼──┼──┤
│  │  │  │  │  │             │  │  │  │  │○ │
└──┴──┴──┴──┴──┘             └──┴──┴──┴──┴──┘

██ = 计算输出 ● 时使用的 3×3 区域
○  = 边界位置（使用了 padding）
```

### 4.2 计算复杂度

对于本例：

- 输入: `[1, 3, 4, 4]` = 48 个数值
- 输出: `[1, 2, 4, 4]` = 32 个数值
- 每个输出需要: $3 \text{通道} \times 3 \times 3 \text{核} = 27$ 次乘法和 26 次加法
- 总乘法次数: $32 \times 27 = 864$ 次
- 总加法次数: $32 \times 26 = 832$ 次

### 4.3 与 1×1 卷积的对比

| 特性   | 1×1 卷积                  | 3×3 卷积                           |
| ---- | ----------------------- | -------------------------------- |
| 感受野  | 1×1                     | 3×3                              |
| 空间信息 | 不聚合                     | 聚合局部信息                           |
| 参数量  | $C_{in} \times C_{out}$ | $9 \times C_{in} \times C_{out}$ |
| 计算量  | 低                       | 高                                |
| 典型用途 | 通道变换、降维/升维              | 特征提取、边缘检测                        |

## 五、BatchNorm2d + SiLU 后续处理

### 5.1 BatchNorm2d 公式

$$
\text{BN}(x) = \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} \times \gamma + \beta
$$

在本例中（演示模式）：

- $\mu = 0$ (running_mean)
- $\sigma^2 = 1$ (running_var)
- $\gamma = 1$ (weight)
- $\beta = 0$ (bias)
- $\epsilon = 10^{-5}$

所以：
$$
\text{BN}(x) = \frac{x}{\sqrt{1 + 10^{-5}}} \approx x
$$

### 5.2 SiLU (Swish) 激活函数

$$
\text{SiLU}(x) = x \times \sigma(x) = \frac{x}{1 + e^{-x}}
$$

### 5.3 计算示例

对于输出通道 0 的位置 (1,1) 的值 (20.4)：

```
卷积后:        20.4000
BatchNorm后:   20.4000 / sqrt(1 + 1e-5) ≈ 20.3999
SiLU后:        20.3999 × sigmoid(20.3999)
              = 20.3999 × 0.999999
              ≈ 20.3999
```

## 六、总结

### 6.1 核心公式回顾

**通用 2D 卷积**:
$$
\text{Out}[n, c_{out}, h, w] = \sum_{c_{in}=0}^{C_{in}-1} \sum_{i=0}^{K_H-1} \sum_{j=0}^{K_W-1} \text{Input}[n, c_{in}, h \cdot S_H + i - P_H, w \cdot S_W + j - P_W] \times \text{Weight}[c_{out}, c_{in}, i, j]
$$

**3×3 卷积 (padding=1)**:
$$
\text{Out}[n, c_{out}, h, w] = \sum_{c_{in}=0}^{C_{in}-1} \sum_{i=0}^{2} \sum_{j=0}^{2} \text{Input}[n, c_{in}, h + i - 1, w + j - 1] \times \text{Weight}[c_{out}, c_{in}, i, j]
$$

### 6.2 关键理解点

1. **3×3 卷积聚合局部空间信息**，每个输出值依赖于 3×3 的输入区域
2. **Padding=1 保持空间尺寸**，使输出与输入具有相同的高度和宽度
3. **边界处理**：使用零填充（zero-padding）处理边界位置
4. **多通道处理**：每个输出通道是所有输入通道卷积结果的累加

### 6.3 代码对应关系

| 数学符号                                   | 代码表示                   | 本例值   |
| -------------------------------------- | ---------------------- | ----- |
| $\text{Input}[n, c, h, w]$             | `x[n, c, h, w]`        | 1~48  |
| $\text{Weight}[c_{out}, c_{in}, i, j]$ | `w[c_out, c_in, i, j]` | 见权重定义 |
